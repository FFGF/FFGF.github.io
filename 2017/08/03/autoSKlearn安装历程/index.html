autoSKlearn安装历程

发表于 2017-08-03 | 分类于 Learn

在ubuntu下安装auto-sklearn

今天在github搜索sklearn时，第一个跳出来的是auto-sklearn看了一下，觉得很神奇，可以自己找到最好的分类器然后自己找到最优化超参数，很感兴趣，就想着自己能不能安装以下，运行一下，搞了一天记录一下历程。

windows下

最开始想着在我自己的主机上进行安装，模拟.pip install auto-sklearn怎么都安装不上，会报错，后来仔细查看了官方的安装教程才发现，坑爹的玩意，不能在windows下直接安装。那就没办法只能在老师给公司买的服务器上面安装耍耍了。

ubuntu下

按照这个教程配置ubuntu下jupyter notebook,不知道为啥我用wget下载anaconda一半会停止，可能是太大了，我就直接去官网下载Linux版本的anaconda,下载的是Anaconda 4.4.0，python3.6为后面的失败埋下伏笔，毕竟auto-sklearn官网说了python>=3.4。然后用WinSCP软件上传到服务器/opt/anaconda下面。
然后用SecureCRTPortable连接远程服务器，cd到/opt/anaconda执行命令 bash bash Anaconda3-4.4.0-Linux-x86_64.sh，安装完毕。这个时候输入python发现还是ubuntu默认的python2.7而不是安装的python3.6，有两种办法解决。

  1. Do you wish the installer to prepend the Anaconda3 install location to PATH in your /root/.bashrc

  2. You may wish to edit your .bashrc or prepend the Anaconda3 install location:$ export PATH=/opt/anaconda3/bin:$PATH
    我按照第二种方法做了之后，关闭远程连接后，再连接远程服务器，输入python发现还是默认的python2.7，第二种可以解决这个问题，原因还不知道。
    按照第一种方法

接下来启动jupyter notebook因为我是root身份登录，所以直接使用jupyter notebook会报错，那就按照提示输入

其他的安装上面说的那个教程就可以了，然后在本地打开http://your ip:8888/就可以了，然后安装官网进行auto-sklearn安装

还有对c++的依赖，统统安装

这个时候提示auto-sklearn安装成功了，开始运行官网例子

直接报错，提示没有classification没有这个模块，可是我去github查看源码确实有，之前还会报一个错，说什么全局变量没定义，google后发现可能是python的版本问题，于是想换成python3.4，说干就干。好在conda很容易创建虚拟环境

然后在这个autoSKlearn安装之前说的包，这个时候输入python提示是python3.4，然后输入import autosklearn.classification没报错，恭喜自己，
下一步是在jupyter notebook中使用这个虚拟环境，查了很多资料后，发现了一个同学写的,按照他说的我并没有找到Libary文件夹，但是给我提供了百度的思路，于是Goole搜索jupyter notebook add virtual environment结果真的google到了运行

然后再次打开

nohup jupyter notebook --allow-root

nohup的作用是关闭连接服务还能一直运行。
然后就可以运行官网的代码了，我以为会很快出结果，结果等了好久，看到官网才发现，要一个多小时，没办法，比较那么多分类器，还有数据预处理外加超参数调优肯定时间很长了。就当玩玩了，实用性还不是很大。

总结

今天本来想跑sklearn里面的人脸识别例子的，结果偶然看到这个，产生兴趣就花了一天的时间，也没白费，配置了jupyter notebook和学会了添加虚拟环境到notebook里面。明天再看人脸识别吧。
